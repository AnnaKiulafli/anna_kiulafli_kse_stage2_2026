# Project reflection

I learned how much work is required to turn a real public dataset into a reproducible analytical pipeline rather than a one-time notebook. Timezone handling mattered because daily counts had to follow Europe/Kyiv calendar days, and administrative levels mattered because oblast, raion, and hromada records cannot be mixed casually. Excluding the partial current day prevented misleading incomplete totals. I deliberately used simple baselines before complex models so the target and validation logic were clear. AI helped accelerate scaffolding, tests, debugging, and documentation, but human checking remained necessary. One concrete review step was catching that `run_eda.py` resolved paths from the current working directory, requesting a repository-root fix, and verifying that the corrected commit was actually added before merging.
